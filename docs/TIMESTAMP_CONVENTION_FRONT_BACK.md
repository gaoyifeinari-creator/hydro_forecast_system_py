# 前后时标统一口径（待确认版）

本文用于统一“数据源时标”和“方案展示时标”的处理规则，避免各链路出现一格偏差。

## 1. 数据源时标（固定事实）

- `hourdb/daydb`（实况雨/流/温）：**后时标**
  - 例：库里 `05:00` 的值，语义是 `04:00~05:00` 时段值。
- `WEA_GFSFORERAIN`（预报降雨）：**前时标**
  - 例：库里 `BTIME=04:00, TIMESPAN=1` 的值，语义是 `04:00~05:00` 时段值。

## 2. 方案展示时标（可配置）

- `dbtype=0`：后时标展示
- `dbtype=-1`：前时标展示

> 目标：界面“所有时间戳”都统一按当前方案展示时标显示。

## 3. 统一处理原则

### 3.1 实况数据（hourdb/daydb）

- 源是后时标。
- 若方案是后时标（`dbtype=0`）：
  - 展示：不转换（库时刻即展示时刻）
- 若方案是前时标（`dbtype=-1`）：
  - 展示：整体 `-1` 步长
  - 例：库 `05:00` -> 界面 `04:00`

### 3.2 预报降雨（WEA_GFSFORERAIN）

- 源是前时标。
- 若方案是前时标（`dbtype=-1`）：
  - 展示：不转换（库时刻即展示时刻）
- 若方案是后时标（`dbtype=0`）：
  - 展示：整体 `+1` 步长
  - 例：库 `04:00` -> 界面 `05:00`

## 4. 起报时间与读库锚点

设：

- `T0_display`：界面展示的“起报时间”（即预报第一个时刻的展示标签）
- `Δt`：方案步长（可为 1h、3h、1day 等）

### 4.1 起报时间规则

- 界面/结果使用的起报时间统一记为 `T0_display`（展示时标下的第一个预报时刻）。

### 4.2 不同来源的读库锚点

- **实况库（后时标源）**
  - `dbtype=0`（后时标展示）：读库锚点 = `T_display`
  - `dbtype=-1`（前时标展示）：读库锚点 = `T_display + Δt`（因为展示时会再 `-Δt`）

- **预报降雨库（前时标源）**
  - `dbtype=-1`（前时标展示）：读库锚点 = `T_display`
  - `dbtype=0`（后时标展示）：读库锚点 = `T_display - Δt`（因为展示时会再 `+Δt`）

## 5. 链路处理清单（需要逐点检查）

1. **起报时间解析**：前端传入 -> `T0_display`（不混入源库时标）
2. **实况读库窗口计算**：按“后时标源 -> 当前展示时标”反推读窗
3. **预报降雨读库窗口计算**：按“前时标源 -> 当前展示时标”反推读窗
4. **实况数据标签映射**：`dbtype=0` 不动；`dbtype=-1` 减一格
5. **预报降雨标签映射**：`dbtype=-1` 不动；`dbtype=0` 加一格
6. **小时->日整编前后顺序**：
   - 先在统一小时画布完成时标映射
   - 再按自然日聚合（或先聚合再统一标签平移，但必须全链路一致）
7. **UI展示一致性**：
   - 图、表、JSON、下载结果、调试日志统一采用同一展示时标
8. **调试输出**：
   - 同时打印 `display_time` 与 `db_read_anchor_time`，便于对账

## 5.1 已落地补充：前时标日方案“尾日漏读”边界

在前时标（`dbtype=-1`）+ 日方案（`Δt=1day`）下，实况站点读取使用了“读窗 +1day、展示标签 -1day”的规则。  
实际排查发现：部分 `daydb` 记录时间戳并非 `00:00`，而是白天时刻（如 `08:00`）。

若实时截断上界 `t_end` 仅平移到当日 `00:00`，会漏掉该“最后一个历史日”的白天记录，表现为：

- 起报时间设为 `11-01` 时，`10-31` 雨量站最后时刻缺失（看起来像“整日无雨量”）。

修复约定（已实现）：

- 当 `dbtype=-1` 且 `time_delta >= 1day` 时，
  - 站点实时读库上界 `station_obs_end` 在原有 `+1day` 基础上再额外 `+1day`（仅放宽上界）；
  - 后续仍通过 `clip_station_dataframe_rows_before_forecast_start(< forecast_start)` 严格裁剪预报段。

这样既能补齐尾日实况，又不会把预报段“伪实况”带入展示/计算。

## 6. 示例（按你给的语义）

假设日方案，`Δt=1day`，当前方案为后时标（`dbtype=0`）：

- 界面显示起报时间：`2025-11-01`（`T0_display`）
- 对应预报降雨读库锚点（前时标源）：`2025-10-31 15:00`（同日当前小时场景）
- 对应实况库读库锚点（后时标源）：`2025-11-01`（不平移）

若同样是 `T0_display=2025-11-01`，但方案改为前时标（`dbtype=-1`）：

- 实况展示需要 `-1day` 标签映射（后时标源 -> 前时标展示）
- 预报降雨展示不需要平移（前时标源 -> 前时标展示）
